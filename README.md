# pxq-auto

票星球监控、飞书控制、多账号登录与 Playwright 创建订单的一体化服务。程序最多创建待支付订单，不执行支付。

## 数据模型

```text
抢票任务（具体场次）
└── 绑定 → 账号、目标数量、票档优先级、观演人

账号（唯一手机号）
└── 独立登录资料，可绑定多个任务
```

- 手机号全局唯一；重复登录会复用原账号和浏览器资料。
- 每个绑定独立保存票档、数量、观演人和订单保护；`UNKNOWN` 只冻结对应绑定。
- 同一账号的多个绑定不会并发执行，并复用该账号已启动的浏览器。
- 解绑或删除任务保留账号；只有 `删除账号` 才删除浏览器资料并释放手机号。
- 预售、在售与回流不需要手动选择模式：开售前进入预抢票，开售且有库存立即抢，开售无库存继续等待回流。

## 代码结构

源码保持单层包结构，按文件职责分组，不引入多层框架目录：

```text
pxq_auto/
├── cli.py                    命令行入口与依赖装配
├── config.py                 系统配置与单账号运行配置
├── db.py                     SQLite 与事务边界
│
├── public_api.py             票星球公开搜索、场次和票档接口
├── task_service.py           任务创建、刷新与展示数据

├── scheduler.py              任务轮询、账号调度与结果通知
│
├── auth.py                   浏览器登录态验证
├── browser.py                Playwright 持久浏览器
├── sale_gate.py              预售等待与开售放行
├── inventory.py              static/dynamic 实时库存
├── seat_selection.py         纯座位组合与评分算法
├── seat_map.py               浏览器座位图操作
├── purchase_page.py          booking 与确认订单页适配
├── order_response.py         创建响应解析与错误分类
├── checkout.py               单账号完整购票工作流
```

内部文件名不属于部署接口；`python -m pxq_auto`、`PXQ_AUTO_DIR`、数据库和账号资料路径均保持不变。

## 安装

```powershell
cd D:\develop\pxq-auto
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e .
python -m playwright install chromium
python -m pxq_auto doctor
```

首次执行会在 `~/.pxq-auto/config.json` 生成配置。填写飞书企业自建应用的 `app_id`、`app_secret`、管理员 open_id 和默认通知群 chat_id。飞书应用需启用机器人、长连接事件 `im.message.receive_v1`、消息收发权限，以及“获取与上传图片或文件资源”权限（登录图形验证码图片需要）。

程序只使用当前数据库结构；升级此版本前请停止服务并删除旧数据库。配置文件不得缺少字段或包含未知字段。

完成 dry-run/真实验证前保持：

```json
"create_order_enabled": false
```

确认后改为 `true`，服务才会放行创建订单请求。无论该值如何，程序都不会执行支付。

## 启动

```powershell
python -m pxq_auto serve
```

服务内同时运行：飞书长连接、指令 worker、任务轮询、登录状态检查和账号抢票 worker。

## 飞书指令

管理员可在群聊或私聊中使用全部指令；非管理员仅可在群聊使用搜索、列表和详情。群聊里的每条消息都需要先 `@机器人`。在群聊登录会公开手机号、验证码和证件信息，请自行决定使用位置。

```text
搜索 <关键词>
抢票 <搜索序号> [间隔 <秒>]
列表
详情 <任务ID>
暂停 <任务ID> / 恢复 <任务ID> / 删除 <任务ID>
间隔 <任务ID> <秒>

登录
账号
删除账号 <账号ID>

绑定 <任务ID> <账号ID>
启动 <任务ID> <账号ID>
停止 <任务ID> <账号ID>
解绑 <任务ID> <账号ID>
```

登录流程：手机号 → 图形验证码图片（需要时）→ 图形验证码 → 短信验证码。登录和绑定的任意步骤都可发送 `取消`；登录会关闭浏览器，新账号还会自动释放手机号，绑定则丢弃本次暂存修改。群聊中回复每一步时仍需 `@机器人`。

同一用户同时只能进行一个登录或绑定流程；分步流程在群聊和私聊之间不会重复开启。

搜索时程序从官方购票须知识别 `无需实名`、`一单一证`、`一票一证`；无法识别时按最严格的一票一证准备。登录成功后发送 `绑定 <任务ID> <账号ID>`，机器人依次引导：

```text
选择逻辑票档 → 设置目标张数 → 选择账号中的官方观演人 → 完成
```

票档按优先级回复编号，例如 `1,3,2`，不提供顺序不明确的“全部”。套票变体不会单独出现，用户只选择基础逻辑票档，票星球在确认订单页自动计算套票与优惠。目标张数范围为 `1~session.limitation`。程序通过账号登录态读取票星球官方观演人，只在飞书展示打码证件号；无需手工录入证件。无需实名不选人；一单一证选一位；一票一证选择与目标张数相同的人数。绑定只在发送 `完成` 后以一个 SQLite 事务整体保存。

`登录` 同时处理新登录和重新登录：新手机号创建账号，已有手机号复用原账号及浏览器资料。

保存绑定后仍处于“已停止”，必须明确发送 `启动 <任务ID> <账号ID>`。同一账号可绑定多个任务，但同一时刻只执行其中一个；每个绑定可以使用不同票档、数量和观演人。同一证件不能同时配置到同一场次的多个账号。

回流检测间隔默认 60 秒，可随时用 `间隔 <任务ID> <秒>` 修改，最低 10 秒。任务的暂停/恢复控制该场次全部绑定；绑定的启动/停止只控制该任务与账号的组合，两者互不覆盖。

订单状态为 `CREATED` 或 `UNKNOWN` 时，人工核对后再次发送对应的 `启动` 即表示允许清除该绑定的订单保护并继续。

## 自动选座与订单安全

- 每个账号的票档严格按飞书输入顺序处理。
- 优先用单一票档满足全部张数；不能满足时，选座演出可跨票档组合，最后才部分购买。
- 非选座演出一单只用一个票档；先寻找能满足全部数量的票档，否则选择可买数量最多的票档部分购买。
- 单张票在最高优先级可售票档内随机选择。
- 多张票依次选择同排连续、同看台最近、全场最近；库存不足时尽量买到更多张。
- `FREE_COMBO` 只用于标记基础票档支持套票，程序不点击套票票档、不计算套票份数或价格。
- 提交前核对票星球确认页的实际票数、票档、实名人数、套票提示和应付金额。
- 座位被抢后重新获取实时库存并重选全部座位。
- 观演人已购时扣除对应目标数量，并按剩余目标重新选择。
- 创建接口未知时仅冻结当前绑定为 `UNKNOWN`，不影响该账号的其他任务。
- 创建请求默认由网络防火墙拦截，每次执行只放行一次；永不点击后续支付。

### 分阶段耗时日志

每次创建尝试都会输出一条 `抢票耗时` 日志。选座任务包含 `dynamic`、票档库存、座位解码、座位评分、选座页、确认页、观演人、提交点击和创建响应；非选座任务输出对应的票档库存与购票页阶段。

`关键路径总计` 从本轮库存查询开始计算，到收到创建响应或明确失败为止；某阶段未出现表示本轮不适用，或资源已经在预热阶段缓存。

```text
抢票耗时｜任务 #1｜账号 #2｜尝试 1｜结果 200｜dynamic=36ms｜票档库存=31ms｜选座页=842ms｜确认页=615ms｜观演人=93ms｜提交点击=21ms｜创建响应=188ms｜创建总耗时=209ms｜关键路径总计=2064ms
```

服务器查看当天的耗时记录：

```bash
journalctl -u pxq-auto --since today -o cat | grep '抢票耗时'
```

## Ubuntu systemd

```ini
[Unit]
Description=pxq-auto
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/pxq-auto
Environment=PXQ_AUTO_DIR=/home/ubuntu/.pxq-auto
ExecStart=/home/ubuntu/pxq-auto/.venv/bin/python -m pxq_auto serve
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

运行数据统一保存在 `PXQ_AUTO_DIR`：配置与 SQLite 位于根目录，每个账号的浏览器资料和订单状态位于 `accounts/<手机号哈希>/`，路径中不出现明文手机号。
