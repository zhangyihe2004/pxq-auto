# pxq-auto

票星球监控、飞书控制、多账号登录与 Playwright 创建订单的一体化服务。程序最多创建待支付订单，不执行支付。

## 数据模型

```text
抢票任务（一个具体场次）
├── 账号 1（唯一手机号）→ 独立浏览器资料 → 独立观演人组
├── 账号 2（唯一手机号）→ 独立浏览器资料 → 独立观演人组
└── 账号 3（唯一手机号）→ 独立浏览器资料 → 独立观演人组
```

- 一个手机号全局只能绑定一个抢票任务。
- 绑定时先用 SQLite 写事务占用手机号，再向票星球发送任何包含手机号的登录请求。
- 每个账号具有独立的登录资料、观演人和订单保护状态；`UNKNOWN` 只冻结对应账号。
- 解绑账号或删除任务会先停止对应 worker，再删除全部账号资料并释放手机号；`CREATED`、`UNKNOWN` 也允许解绑。
- 预售、在售与回流不需要手动选择模式：开售前进入预抢票，开售且有库存立即抢，开售无库存继续等待回流。

## 安装

```powershell
cd D:\develop\pxq-auto
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e .
python -m playwright install chromium
python -m pxq_auto doctor
```

首次执行会在 `~/.pxq-auto/config.json` 生成配置。填写飞书企业自建应用的 `app_id`、`app_secret`、管理员 open_id 和默认通知群 chat_id。飞书应用需启用机器人、长连接事件 `im.message.receive_v1`、消息收发权限，以及“获取与上传图片或文件资源”权限（登录图形验证码图片需要）。

当前版本只支持全新数据库和完整的当前版 `config.json`，不包含旧表结构或旧配置字段迁移。重新安装时请使用新的 `PXQ_AUTO_DIR`，不要复制旧数据库；配置文件不得缺少字段或包含未知字段。

完成 dry-run/真实验证前保持：

```json
"create_order_enabled": false
```

确认后改为 `true`，服务才会放行创建订单请求。无论该值如何，程序都不会执行支付。

## 启动

```powershell
python -m pxq_auto serve
```

服务内同时运行：飞书长连接、指令 worker、任务轮询、登录状态检查和账号抢票 worker。

## 飞书指令

管理员可在群聊或私聊中使用全部指令；非管理员仅可在群聊使用搜索、列表和详情。群聊里的每条消息都需要先 `@机器人`。在群聊登录会公开手机号、验证码和证件信息，请自行决定使用位置。

```text
搜索 <关键词>
抢票 <搜索序号> [间隔 <秒>]
列表
详情 <任务ID>
暂停 <任务ID> / 恢复 <任务ID> / 删除 <任务ID>
间隔 <任务ID> <秒>

登录 <任务ID>
配置 <账号ID>
启动 <账号ID>
停止 <账号ID>
解绑 <账号ID>
重置 <账号ID>
```

登录流程：手机号 → 图形验证码图片（需要时）→ 图形验证码 → 短信验证码。登录和配置的任意步骤都可发送 `取消`；登录会关闭浏览器，新账号还会自动释放手机号，配置则丢弃本次暂存修改。群聊中回复每一步时仍需 `@机器人`。

同一用户同时只能进行一个登录或配置流程；分步流程在群聊和私聊之间不会重复开启。

登录成功后发送 `配置 <账号ID>`，机器人依次引导选择票档和逐个添加观演人。票档必须按优先级回复编号，例如 `1,3,2`，不提供顺序不明确的“全部”。观演人格式是 `姓名 身份证前3位 后4位`，例如 `张三 210 2534`；程序会在内部补成票星球使用的打码形式。全部添加后发送 `完成`。配置只在“完成”时以一个 SQLite 事务整体保存，中途输入不会影响正式配置。

`登录 <任务ID>` 同时处理新登录和重新登录：新手机号会创建账号；已经绑定当前任务的手机号会复用原账号及浏览器资料；已绑定其他任务的手机号会被拒绝。

保存配置后账号仍处于“已停止”，必须明确发送 `启动 <账号ID>` 才会抢票。任何登录、添加第一位观演人或修改票档都不会隐式启动。修改运行账号的配置会先停止该账号，修改完成后需要重新启动。每个账号可以使用不同票档和观演人；同一证件不能同时配置到同一场次的多个账号。

回流检测间隔默认 60 秒，可随时用 `间隔 <任务ID> <秒>` 修改，最低 10 秒。任务的暂停/恢复控制该场次全部账号；账号的启动/停止只控制单个账号，两者互不覆盖。

`重置` 只应在已经人工确认并取消待支付订单后使用。重置后账号仍是“已停止”，需要再次发送 `启动 <账号ID>`。

## 自动选座与订单安全

- 每个账号的票档严格按飞书输入顺序处理。
- 单张票在最高优先级可售票档内随机选择。
- 多张票依次选择同排连续、同看台最近、全场最近；库存不足时尽量买到更多张。
- 座位被抢后重新获取实时库存并重选全部座位。
- 观演人已购时移除冲突观演人并按剩余人数重选。
- 创建接口未知时仅冻结当前账号为 `UNKNOWN`，禁止该账号自动重试。
- 创建请求默认由网络防火墙拦截，每次执行只放行一次；永不点击后续支付。

## Ubuntu systemd

```ini
[Unit]
Description=pxq-auto
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/pxq-auto
Environment=PXQ_AUTO_DIR=/home/ubuntu/.pxq-auto
ExecStart=/home/ubuntu/pxq-auto/.venv/bin/python -m pxq_auto serve
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

运行数据统一保存在 `PXQ_AUTO_DIR`：配置与 SQLite 位于根目录，每个账号的浏览器资料和订单状态位于 `accounts/<手机号哈希>/`，路径中不出现明文手机号。
